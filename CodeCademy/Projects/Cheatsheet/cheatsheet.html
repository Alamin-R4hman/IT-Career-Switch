Name Tag Description Notes
Em em em represents the font-size of the current element or the default base font-size set by the browser if none is given If the base font of a browser is 16 pixels (which is normally the default size of text in a browser), then 1 em is equal to 16 pixels. 2 ems would equal 32 pixels, and so on.
Rem rem Rem stands for root em. It acts similar to em, but instead of checking parent elements to size font, it checks the root element. The root element is the html tag. Most browsers set the font size of html to 16 pixels, so by default rem measurements will be compared to that value. To set a different font size for the root element, you can add a CSS rule like html {font-size: 20px;}.
Property Value Description Notes
Property Value Description Notes
Display :inline-flex Allows multiple flex containers to appear inline next to each other. inline-flex makes an element a flex container, like display: flex, but it also makes the element behave like an inline element. This means the flex container will align with other inline elements on the same line if there’s enough space.
Display :inline Behave's like an inline element. If you use display: inline instead of display: inline-flex, the element will behave like a regular inline element. It will not become a flex container, so its children won’t be flex items. This means you won’t be able to use flexbox properties
Display :flex display: flex; on a container element makes it a flex container but will behave as a block-level container which means it will take full widht of container. It wont sit inline with other elements. This changes how its child elements (flex items) are laid out. The main effects are:
Horizontal Layout: Child elements are placed in a row by default.
Alignment: You can easily align child elements along the main axis (row) and cross axis (column).
Space Distribution: Child elements can grow, shrink, and be spaced out automatically to fill the container.
Flex --- Flex Properties and what they do flex-basis:sets the initial size of a flex item before any space distribution happens.
flex-grow: Allows the item to grow if there’s extra space in the container.
flex-shrink: Allows the item to shrink if there’s not enough space in the container.
Justify Content justify-content: Spaces items along the main axis (horizontally, usually) Can use the following values:
flex-start;, center;, flex-end;, space-between;, space-around;, space-evenly;
Justify Content justify-content: flex-start;, center;, flex-end;, space-between;, space-around;, space-evenly;
Align Items align-items: Spaces items along the cross axis (vertically, usually).

Name Description Notes
Number Any number, including numbers with decimals: 4, 8, 1516, 23.42.
BigInt Any number, greater than 253-1 or less than -(253-1), with n appended to the number: 1234567890123456n.
String Any grouping of characters on your keyboard (letters, numbers, spaces, symbols, etc.) surrounded by single quotes: ' ... ' or double quotes " ... ", though we prefer single quotes. Some people like to think of string as a fancy word for text.
Boolean This data type only has two possible values— either true or false (without quotes). It’s helpful to think of booleans as on and off switches or as the answers to a “yes” or “no” question.
Null This data type represents the intentional absence of a value, and is represented by the keyword null (without quotes).
Undefined This data type is denoted by the keyword undefined (without quotes). It also represents the absence of a value though it has a different use than null. undefined means that a given value does not exist.
Symbol A newer feature to the language, symbols are unique identifiers, useful in more complex coding. No need to worry about these for now.
Object Collections of related data.
